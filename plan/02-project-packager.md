# 模組 2：專案打包器

## 目的

把使用者寫的遊戲程式碼和資源（圖片、音效、字型等）收集起來，打包成瀏覽器能下載的檔案。

## 要做什麼

使用者的遊戲專案目錄裡有 `.py` 檔，有 `resources/` 目錄裡的圖片音效。這個模組要：

1. 走訪目錄，找出所有需要的檔案
2. 跳過不需要的東西（`.git`、`__pycache__`、虛擬環境等）
3. 把這些檔案打成一個壓縮包（tar.gz）

## 怎麼做

瀏覽器載入 wasm 之後，wasm 裡面有一個虛擬檔案系統（MEMFS）。Python 直譯器跑在 wasm 裡，它讀檔案其實是讀這個虛擬檔案系統。所以遊戲的 `.py` 檔和資源檔必須被放進這個虛擬檔案系統裡。

兩種放法：

### 開發時：Runtime fetch

把檔案打成 tar.gz，瀏覽器先下載這個 tar.gz，然後用 JS 解壓後一個一個寫進虛擬檔案系統。

好處是改了程式碼只要重新打包 tar.gz 就好，幾秒鐘的事。

### 發佈時：Emscripten preload

用 Emscripten 的打包工具 `file_packager.py` 把遊戲檔案嵌進一個 `.data` 檔，啟動時自動載入。

好處是快，壞處是每次改動都要重新打包。
